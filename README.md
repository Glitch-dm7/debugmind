Paste an error, find matching fixes from past bugs across all your projects. Powered by a Cognee knowledge graph with vector + graph memory.

---

## Quick Start

```bash
# Start the Cognee knowledge graph engine
docker compose up -d

# Start the backend (port 3001)
cd backend && bun install && bun run dev

# In another terminal ‚ÄĒ start the frontend (port 5173)
cd frontend && bun install && bun run dev
```

Seed some sample data:

```bash
cd backend && bun run seed
```

Open [localhost:5173](http://localhost:5173) and paste a stack trace.

## Architecture

| Layer | Stack | Role |
|-------|-------|------|
| Frontend | React 19 + Vite 8 + Tailwind v4 | SPA for searching and logging bugs |
| Backend | Hono 4 + Bun runtime | REST API, business logic, Cognee integration |
| Knowledge Graph | Cognee + LanceDB + SQLite | Semantic/vector search + entity relationships |
| Local Store | `bugs.json` | Structured metadata for fast fingerprint lookups |

---

## How It Works

1. **Log a bug** ‚ÄĒ paste an error + fix. The error is normalized (strips timestamps, addresses, paths), fingerprinted (MD5), and stored in Cognee + `bugs.json`.

2. **Search for a bug** ‚ÄĒ paste a new error. First checks for an exact fingerprint match (instant). If none found, queries Cognee's knowledge graph via `GRAPH_COMPLETION` for semantic similarity.

3. **Results ranked** ‚ÄĒ each result is scored by `similarity √ó Wilson score`. Wilson's lower bound penalizes low sample sizes so a 50-confirmed fix ranks higher than a single lucky guess.

4. **Feedback** ‚ÄĒ mark a fix as "worked" or "didn't apply". Confidence updates are factored into future searches.

## Confidence Scoring

Each fix is ranked by `displayScore = similarity √ó wilsonScoreLowerBound`:

| Feedback | Naive ratio | Wilson score |
|----------|-------------|--------------|
| 1‚úď / 1   | 100%        | 21%          |
| 5‚úď / 5   | 100%        | 57%          |
| 50‚úď / 50 | 100%        | 93%          |

Wilson's lower bound (95% CI) prevents overconfidence on small sample sizes.
